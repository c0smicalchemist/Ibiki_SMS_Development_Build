  ‚úÖ NEW ENDPOINT
================================================================================

METHOD: GET
URL: http://151.243.109.79/api/v2/sms/messages

DESCRIPTION:
  Retrieve ALL messages sent by your account with their status information.
  Returns up to 100 messages by default.

AUTHENTICATION:
  Requires API key in Authorization header (same as other endpoints)

QUERY PARAMETERS:
  - limit (optional): Number of messages to return (default: 100)
    Example: ?limit=50

================================================================================
  üìù HOW TO USE
================================================================================

BASIC REQUEST (Get last 100 messages):
```bash
curl -X GET "http://151.243.109.79/api/v2/sms/messages" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

REQUEST WITH LIMIT (Get last 50 messages):
```bash
curl -X GET "http://151.243.109.79/api/v2/sms/messages?limit=50" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

GET SPECIFIC NUMBER OF MESSAGES:
```bash
curl -X GET "http://151.243.109.79/api/v2/sms/messages?limit=20" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

================================================================================
  üìä RESPONSE FORMAT
================================================================================

SUCCESS RESPONSE:
```json
{
  "success": true,
  "messages": [
    {
      "id": "msg_abc123",
      "messageId": "60f1a5b3e6e7c12345678901",
      "endpoint": "sendsingle",
      "recipient": "+1234567890",
      "recipients": null,
      "status": "delivered",
      "totalCost": "0.05",
      "totalCharge": "0.10",
      "messageCount": 1,
      "createdAt": "2025-11-18T10:30:00.000Z",
      "requestPayload": "{...}",
      "responsePayload": "{...}"
    },
    {
      "id": "msg_def456",
      "messageId": "60f1a5b3e6e7c12345678902",
      "endpoint": "sendbulk",
      "recipient": null,
      "recipients": ["+1234567890", "+0987654321"],
      "status": "sent",
      "totalCost": "0.10",
      "totalCharge": "0.20",
      "messageCount": 2,
      "createdAt": "2025-11-18T11:00:00.000Z",
      "requestPayload": "{...}",
      "responsePayload": "{...}"
    }
  ],
  "count": 2,
  "limit": 100
}
```

RESPONSE FIELDS:
  - success: true if request succeeded
  - messages: Array of all sent messages
    - id: Internal message log ID
    - messageId: ExtremeSMS message ID (use this to check detailed status)
    - endpoint: Which API endpoint was used (sendsingle, sendbulk, etc)
    - recipient: Single recipient (for sendsingle)
    - recipients: Multiple recipients (for sendbulk/sendbulkmulti)
    - status: Current message status (queued, sent, delivered, failed)
    - totalCost: What you paid to ExtremeSMS
    - totalCharge: What your client was charged
    - messageCount: Number of messages sent
    - createdAt: When the message was sent (ISO 8601 format)
    - requestPayload: Original request data (JSON string)
    - responsePayload: ExtremeSMS response (JSON string)
  - count: Total number of messages returned
  - limit: The limit that was applied

================================================================================
  üÜö COMPARISON: OLD vs NEW
================================================================================

OLD WAY (Check ONE message):
  Endpoint: GET /api/v2/sms/status/{messageId}
  Limitation: You need to know the specific message ID
  Use Case: Check status of one specific message
  
  Example:
  curl -X GET "http://151.243.109.79/api/v2/sms/status/60f1a5b3e6e7c12345678901" \
    -H "Authorization: Bearer YOUR_API_KEY"

NEW WAY (Check ALL messages):
  Endpoint: GET /api/v2/sms/messages
  Advantage: Get ALL messages at once
  Use Case: Monitor all messages, find message IDs, audit trail
  
  Example:
  curl -X GET "http://151.243.109.79/api/v2/sms/messages?limit=100" \
    -H "Authorization: Bearer YOUR_API_KEY"

BOTH STILL WORK:
  - Use NEW endpoint to get all messages
  - Use OLD endpoint to get detailed status of specific message
